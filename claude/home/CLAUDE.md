# Global Instructions

## コミュニケーション

- 応答言語: 日本語
- コード、コミットメッセージ、ブランチ名は英語で書く

## Git ワークフロー

- コミットメッセージは Conventional Commits 形式（`feat:`, `fix:`, `docs:`, `refactor:`, `test:`, `chore:` 等）

## コーディングスタイル

- 既存コードのスタイルに従う（新しい規約を勝手に導入しない）
- リファクタリングは許可するが、機能変更とリファクタリングはコミットを分けること

## 推測による実装のルール

- 不明点があっても推測で実装して進めてよい
- ただし以下を必ず行う:
  - 該当コード内に `// ASSUMPTION:` コメントで推測箇所を明記
  - プロジェクトルートに `ASSUMPTIONS.md` を作成/追記し、以下を記録:
    - 何の情報が不足していたか
    - どのように推測したか
    - 推測に基づいて実装した箇所

## 品質・安全性

- 変更後は可能な限りリンター/テストを実行して確認する
- 機密情報（APIキー、トークン等）をコードに含めない

name: daily-narrative
description: デイリーノートからその日の出来事と感情に重きを置いた振り返りナラティブを生成する
---

## 指示

指定された日付（デフォルト: 当日）のデイリーノートを読み取り、その日の振り返りを散文形式で生成してください。

対象日付: $ARGUMENTS（未指定の場合は今日の日付を使用）

### 入力

- `daily/<日付>.md` の `## memo` を **除く** 全セクション（`## work`, `## tech`, `## private`, `## book`）

### 出力

- `daily-narrative/<日付>.md`

### 手順

1. `$ARGUMENTS` が未指定の場合は `date +%Y-%m-%d` で当日の日付を取得する
2. `daily/<日付>.md` の存在を確認する。存在しない場合はその旨を伝えて終了する
3. ファイルを読み取り、`## memo` を除く全セクション（work, tech, private, book）の内容を収集する
4. 収集した内容から以下を抽出・特定する:
   - ユーザーが明示的に感情を記載している箇所（例:「楽しかった」「疲れた」「嬉しい」等）
   - 気持ちの変動があったと推測される事象（例: 新しい技術への挑戦、トラブル対応、達成・完了、予定変更など）
5. 特定した事象について、ユーザーに対話的に質問する:
   - 明示的な感情記載がある箇所はそのまま採用する（確認不要）
   - 気持ちの変動が推測される事象については「このとき、どんな気持ちでしたか？」等の形で質問する
   - ユーザーの回答を収集する
6. 収集した内容＋ユーザーの感情回答をもとに、以下のスタイルガイドに従ってナラティブを生成する
7. `daily-narrative/<日付>.md` に出力する
   - ファイルが既に存在し、同じ日付のヘッダー（`## <日付>`）が含まれている場合は上書きしない旨を伝えて終了する

### スタイルガイド

- **視点**: 一人称（「〜した」「〜と感じた」）
- **形式**: 散文形式。箇条書きは使わない
- **流れ**: 時系列に沿った自然な流れで、出来事・学びに加え、ユーザーから収集した感情・内省を織り交ぜる
  - 作業時間の記録（`⏱ 開始`/`⏱ 終了`）があればそれを時系列の手がかりにする
  - 複数セクションの内容を自然に繋げる（セクションごとに分けず、一つの物語として構成する）
- **文字数目安**: 300〜800字
- **トーン**: 内省的で落ち着いた語り口。日記のような親しみやすさ

### 出力フォーマット

```markdown
## <日付>

（ナラティブ本文）
```

### 注意事項

- 事実の捏造はしない。デイリーノートに書かれていない出来事を追加しない
- 感情の特定（気持ちの変動がありそうだという推測）は行ってよいが、実際の感情はユーザーの明示的な記載または対話での回答に基づくこと
- AIが勝手に感情を創作・補完しないこと
- `## memo` セクションの内容はナラティブに含めない
- 既存ファイルに同じ日付の内容がある場合は重複追記しない

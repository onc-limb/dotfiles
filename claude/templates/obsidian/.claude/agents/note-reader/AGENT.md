---

呼び出し元の指示に従い、指定されたファイルを読み取り、構造化されたデータを返してください。

## 手順

1. 呼び出し元が指定したファイルパターンで Glob を実行し、対象ファイル一覧を取得する
2. 全てのファイルを Read で読み取る（スキップ不可）
3. 呼び出し元が指定した収集項目に従い、各セクションからデータを抽出する
4. 構造化された形式で結果を返す

## 読み取り対象のファイル形式

### 日報ファイル（`daily-report/YYYY-MM-DD.md`）

```markdown
## 稼働時間

| 項目 | 時刻 |
|---|---|
| 開始 | HH:MM |
| 終了 | HH:MM |
| 休憩 | X時間XX分 |
| **稼働合計** | **X時間XX分** |

## 作業内容

### プロジェクト名
- 作業項目（時間帯があれば記載）

## 進捗・成果
- 成果物の記載

## 課題・ブロッカー
- 課題の記載

## 翌営業日の予定
- 予定の記載

## 連絡・相談事項
- 相談事項の記載
```

### デイリーノート（`daily/YYYY-MM-DD.md`）

```markdown
## work

### #ref/プロジェクト名
⏱ 開始 HH:MM
- 作業項目
⏸ 休憩開始 HH:MM
⏸ 休憩終了 HH:MM
- 作業項目
⏱ 終了 HH:MM

## tech
### #ref/技術名
- 内容

## private
### #ref/トピック名
- 内容

## memo
- メモ
```

## 出力形式

呼び出し元の指示に応じた形式で返す。以下は月次報告書向けの典型例:

```
## 読み取り結果: YYYY年MM月

### 日別稼働データ
- YYYY-MM-DD: 開始 HH:MM / 終了 HH:MM / 休憩 X時間XX分 / 稼働 X時間XX分
- YYYY-MM-DD: 開始 HH:MM / 終了 HH:MM / 休憩 X時間XX分 / 稼働 X時間XX分
...

### プロジェクト別作業内容
#### プロジェクト名A
- MM/DD: 作業内容
- MM/DD: 作業内容
...

#### プロジェクト名B
- MM/DD: 作業内容
...

### 成果・納品物
- MM/DD: 成果の内容
- MM/DD: 成果の内容
...

### 課題・ブロッカー
- MM/DD: 課題の内容（解決済み / 未解決）
...

### 翌月の参考情報
- 最終日報の翌営業日予定の内容
```

## 注意事項

- 対象ファイルは全て漏れなく読み取ること
- データは加工・要約せずそのまま抽出する（集計・要約は呼び出し元の責務）
- 対象ファイルが存在しない場合は、その旨を報告する
